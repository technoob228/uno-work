## Главный вывод по приложению

На записи зафиксирована проблема UX/функциональности: пользователь ожидает, что ассистент сможет посмотреть скриншот или логотип, но приложение/модель отвечает, что image input не поддерживается.

Возможные причины:

- выбрана модель без поддержки vision/image input;
- UI не показывает заранее, что текущая модель не умеет читать изображения;
- вложение изображения не передаётся в provider runtime;
- provider capability detection неверно определяет возможности модели;
- fallback-сообщение слишком общее и не предлагает переключиться на vision-capable модель.

## Что можно улучшить в Uno Work

1. Показывать в UI, поддерживает ли выбранная модель изображения.
2. Если пользователь прикрепил/передал изображение, а модель не vision-capable, предлагать переключение модели.
3. Не давать отправить image-only запрос в модель без image input без явного предупреждения.
4. Добавить понятное сообщение: какая текущая модель выбрана и какие модели могут прочитать изображение.
5. Проверить pipeline вложений: доходит ли изображение от web UI до server/provider layer.
